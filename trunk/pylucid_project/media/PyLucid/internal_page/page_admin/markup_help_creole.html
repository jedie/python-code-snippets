Creole Additions/Macros

  • source code:
    You can insert sourcecode: Use <<code ext=.EXT>>...<</code>> and PyLucid used Pygments (if available) to highlight it.
    '.EXT' is the typical fileextension/alias of the code.

    example:

    <<code ext=.css>>
/* CSS Stylesheet */
.xs {font-family:verdana,arial,helvetica,sans-serif;font-size: x-small}
.m {font-size: medium}
<</code>>
  • html code:
    You can insert html code. But it will only pass-through without extra decoration if all lines starts and ends with a normal html tag! Otherwise the code would be escaped.
    There exist also a macro to pass everything throught: <<html>>...<</html>>

Changes to the original creole specs:

  • image tag:
    We implemented a more stricted image tag than the original, because it should not match on django template tags!
    It matches only if...

    1. ...image target ends with a picture extention
    2. ...separator >|< and the image text exist
  • line break:
    We used blog-style line break insteat of wiki-style (every \n would be convertet into <br />)
  • django tags/blocktags:
    You can insert django tags and blocktags. No extra decoration is not needed.
